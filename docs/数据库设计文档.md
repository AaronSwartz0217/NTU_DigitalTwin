# 数据库设计文档

> **版本: 2.1.0 (将近完成版)** | 更新日期: 2026-06-13

## 概述

本文档定义南通大学启东校区数字孪生应用的数据库设计。应用采用**本地缓存 + 后端同步 + SharedPreferences 辅助缓存**的架构模式：

| 数据库类型 | 用途 | 技术栈 |
|------------|------|--------|
| **本地数据库** | 用户/帖子/评论/消息/天气历史 本地缓存 | SQLite + Room ORM |
| **SharedPreferences** | 聊天消息缓存、认证配置、用户资料缓存 | Android 原生存储 |
| **后端数据库** | 核心数据存储、多端同步、用户权限管理 | MySQL + SqlSugar |

### 本地数据库（Android 客户端）
**数据库名称**: `ntu_qidong.db`
**版本**: 2.0
**目标 SDK**: Android 10+ (API 29)
**ORM 框架**: Room

### 后端数据库（Furion 服务端）
**数据库名称**: `chat_db`
**数据库类型**: MySQL
**ORM 框架**: SqlSugar

---

## 数据库架构

### 本地数据库（SQLite）

```
ntu_qidong.db
```

## 表结构详情

### 1. user (用户信息表)

存储用户基本信息。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 用户ID |
| user_name | TEXT | NOT NULL, UNIQUE | 用户名（登录用） |
| password | TEXT | - | 密码（仅本地用户使用） |
| name | TEXT | - | 真实姓名 |
| no | TEXT | - | 学号/工号 |
| avatar | TEXT | - | 头像URL |
| gender | INTEGER | DEFAULT 0 | 性别 (0=男, 1=女) |
| email | TEXT | - | 邮箱 |
| phone | TEXT | - | 手机号 |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| updated_at | INTEGER | NOT NULL | 更新时间戳 |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS user (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_name TEXT NOT NULL UNIQUE,
    password TEXT,
    name TEXT,
    no TEXT,
    avatar TEXT,
    gender INTEGER DEFAULT 0,
    email TEXT,
    phone TEXT,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

#### 索引

- `idx_user_user_name`: user_name 字段索引

#### 示例数据

| id | user_name | password | name | no | avatar | gender | created_at | updated_at |
|----|-----------|----------|------|----|--------|--------|------------|------------|
| 1 | admin | - | 管理员 | 00001 | - | 0 | 1716192000000 | 1716192000000 |

### 2. post (帖子表)

存储论坛帖子信息。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 帖子ID |
| title | TEXT | NOT NULL | 帖子标题 |
| content | TEXT | NOT NULL | 帖子内容 |
| author_id | INTEGER | NOT NULL, FOREIGN KEY | 作者ID |
| category | TEXT | - | 分类 |
| likes | INTEGER | DEFAULT 0 | 点赞数 |
| comments | INTEGER | DEFAULT 0 | 评论数 |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| updated_at | INTEGER | NOT NULL | 更新时间戳 |
| is_deleted | INTEGER | DEFAULT 0 | 是否删除 |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS post (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    author_id INTEGER NOT NULL,
    category TEXT,
    likes INTEGER DEFAULT 0,
    comments INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,
    is_deleted INTEGER DEFAULT 0,
    FOREIGN KEY (author_id) REFERENCES user(id) ON DELETE CASCADE
);
```

#### 索引

- `idx_post_author_id`: author_id 字段索引
- `idx_post_created_at`: created_at 字段索引

### 3. comment (评论表)

存储帖子评论信息。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 评论ID |
| post_id | INTEGER | NOT NULL, FOREIGN KEY | 帖子ID |
| author_id | INTEGER | NOT NULL, FOREIGN KEY | 作者ID |
| content | TEXT | NOT NULL | 评论内容 |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| is_deleted | INTEGER | DEFAULT 0 | 是否删除 |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS comment (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    post_id INTEGER NOT NULL,
    author_id INTEGER NOT NULL,
    content TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    is_deleted INTEGER DEFAULT 0,
    FOREIGN KEY (post_id) REFERENCES post(id) ON DELETE CASCADE,
    FOREIGN KEY (author_id) REFERENCES user(id) ON DELETE CASCADE
);
```

#### 索引

- `idx_comment_post_id`: post_id 字段索引
- `idx_comment_author_id`: author_id 字段索引

### 4. message (消息表)

存储聊天消息信息。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 消息ID |
| sender_id | INTEGER | NOT NULL, FOREIGN KEY | 发送者ID |
| receiver_id | INTEGER | NOT NULL, FOREIGN KEY | 接收者ID |
| content | TEXT | NOT NULL | 消息内容 |
| type | INTEGER | DEFAULT 0 | 消息类型 (0=文本, 1=图片, 2=语音) |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| is_read | INTEGER | DEFAULT 0 | 是否已读 |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS message (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sender_id INTEGER NOT NULL,
    receiver_id INTEGER NOT NULL,
    content TEXT NOT NULL,
    type INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL,
    is_read INTEGER DEFAULT 0,
    FOREIGN KEY (sender_id) REFERENCES user(id) ON DELETE CASCADE,
    FOREIGN KEY (receiver_id) REFERENCES user(id) ON DELETE CASCADE
);
```

#### 索引

- `idx_message_sender_id`: sender_id 字段索引
- `idx_message_receiver_id`: receiver_id 字段索引
- `idx_message_created_at`: created_at 字段索引

### 5. weather_history (天气历史记录表)

存储每日天气数据，用于趋势分析和图表展示。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| date | TEXT | NOT NULL, UNIQUE | 日期 (格式: yyyy-MM-dd) |
| temp_avg | REAL | NOT NULL | 日平均气温 (℃) |
| temp_max | REAL | NOT NULL | 日最高气温 (℃) |
| temp_min | REAL | NOT NULL | 日最低气温 (℃) |
| humidity | INTEGER | NOT NULL | 相对湿度 (%) |
| weather_condition | TEXT | - | 天气状况 |
| wind_direction | TEXT | - | 风向 |
| wind_speed | REAL | - | 风速 |
| feels_like | REAL | - | 体感温度 |
| created_at | INTEGER | NOT NULL | 创建时间戳 (Unix timestamp) |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS weather_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL UNIQUE,
    temp_avg REAL NOT NULL,
    temp_max REAL NOT NULL,
    temp_min REAL NOT NULL,
    humidity INTEGER NOT NULL,
    weather_condition TEXT,
    wind_direction TEXT,
    wind_speed REAL,
    feels_like REAL,
    created_at INTEGER NOT NULL
);
```

#### 索引

- `idx_weather_history_date`: date 字段索引

### 6. app_visit (应用访问记录表)

记录应用每日启动访问次数，用于统计分析。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| visit_date | TEXT | NOT NULL, UNIQUE | 访问日期 (格式: yyyy-MM-dd) |
| visit_count | INTEGER | NOT NULL, DEFAULT 0 | 当日访问次数 |
| last_visit_time | INTEGER | NOT NULL | 最后访问时间戳 (Unix timestamp) |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS app_visit (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    visit_date TEXT NOT NULL UNIQUE,
    visit_count INTEGER NOT NULL DEFAULT 0,
    last_visit_time INTEGER NOT NULL
);
```

#### 索引

- `idx_app_visit_date`: visit_date 字段索引

## 数据完整性约束

1. **UNIQUE 约束**: date、visit_date、user_name 字段确保唯一性
2. **NOT NULL 约束**: 关键字段不允许为空
3. **DEFAULT 值**: 部分字段设置默认值
4. **外键约束**: 启用外键关联，支持级联删除
5. **Room 注解**: 使用 @Entity、@PrimaryKey、@ForeignKey 等注解
