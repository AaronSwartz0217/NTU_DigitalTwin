# 数据库设计文档

## 概述

本文档定义南通大学启东校区数字孪生应用的 SQLite 数据库设计。

**数据库名称**: `ntu_qidong.db`
**版本**: 1.0
**目标 SDK**: Android 10+ (API 29)

---

## 数据库架构

```
ntu_qidong.db
├── weather_history (天气历史记录表)
└── app_visit (应用访问记录表)
```

---

## 表结构详情

### 1. weather_history (天气历史记录表)

存储每日天气数据，用于趋势分析和图表展示。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| date | TEXT | NOT NULL, UNIQUE | 日期 (格式: yyyy-MM-dd) |
| temp_avg | REAL | NOT NULL | 日平均气温 (℃) |
| temp_max | REAL | NOT NULL | 日最高气温 (℃) |
| temp_min | REAL | NOT NULL | 日最低气温 (℃) |
| humidity | INTEGER | NOT NULL | 相对湿度 (%) |
| weather_condition | TEXT | - | 天气状况 |
| created_at | INTEGER | NOT NULL | 创建时间戳 (Unix timestamp) |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS weather_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL UNIQUE,
    temp_avg REAL NOT NULL,
    temp_max REAL NOT NULL,
    temp_min REAL NOT NULL,
    humidity INTEGER NOT NULL,
    weather_condition TEXT,
    created_at INTEGER NOT NULL
);
```

#### 索引

- `idx_weather_history_date`: date 字段索引，用于按日期查询

```sql
CREATE INDEX IF NOT EXISTS idx_weather_history_date ON weather_history(date);
```

#### 示例数据

| id | date | temp_avg | temp_max | temp_min | humidity | weather_condition | created_at |
|----|------|----------|----------|----------|----------|-------------------|------------|
| 1 | 2024-05-20 | 23.5 | 28.0 | 19.0 | 65 | 多云 | 1716192000000 |
| 2 | 2024-05-21 | 24.0 | 29.0 | 20.0 | 60 | 晴 | 1716278400000 |

### 2. app_visit (应用访问记录表)

记录应用每日启动访问次数，用于统计分析。

#### 字段定义

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| visit_date | TEXT | NOT NULL, UNIQUE | 访问日期 (格式: yyyy-MM-dd) |
| visit_count | INTEGER | NOT NULL, DEFAULT 0 | 当日访问次数 |
| last_visit_time | INTEGER | NOT NULL | 最后访问时间戳 (Unix timestamp) |

#### 建表 SQL

```sql
CREATE TABLE IF NOT EXISTS app_visit (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    visit_date TEXT NOT NULL UNIQUE,
    visit_count INTEGER NOT NULL DEFAULT 0,
    last_visit_time INTEGER NOT NULL
);
```

#### 索引

- `idx_app_visit_date`: visit_date 字段索引

```sql
CREATE INDEX IF NOT EXISTS idx_app_visit_date ON app_visit(visit_date);
```

#### 示例数据

| id | visit_date | visit_count | last_visit_time |
|----|------------|-------------|-----------------|
| 1 | 2024-05-20 | 5 | 1716192000000 |
| 2 | 2024-05-21 | 8 | 1716278400000 |

## 数据操作接口

### DAO 层类

#### WeatherHistoryDAO

提供 `weather_history` 表的 CRUD 操作:

| 方法 | 说明 |
|------|------|
| `insertOrUpdate(WeatherHistory)` | 插入或更新历史记录 |
| `findByDate(String date)` | 按日期查询 |
| `findRecentDays(int days)` | 获取最近 N 天数据 |
| `findAll()` | 获取所有数据 |

#### AppVisitDAO

提供 `app_visit` 表的 CRUD 操作:

| 方法 | 说明 |
|------|------|
| `recordVisit(String visitDate)` | 记录一次访问 |
| `getTotalVisitCount()` | 获取累计访问次数 |
| `getTodayVisitCount(String visitDate)` | 获取今日访问次数 |
| `findByDate(String visitDate)` | 按日期查询 |
| `findAll()` | 获取所有记录 |

---

## 数据完整性约束

1. **UNIQUE 约束**: date 和 visit_date 字段确保每日只有一条记录
2. **NOT NULL 约束**: 关键字段不允许为空
3. **DEFAULT 值**: visit_count 默认为 0
4. **外键约束**: 已启用 `setForeignKeyConstraintsEnabled(true)`
